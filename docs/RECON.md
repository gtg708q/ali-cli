# Alibaba API Reconnaissance

*Reference documentation for Alibaba's buyer-side APIs, captured via
Playwright `page.evaluate()` from live sessions.*

> **Note:** Example response bodies below contain real account IDs, supplier
> names, and conversation IDs captured during development. They are
> illustrative only — your own session will return your own values. They
> are not credentials and do not grant access to any account.

## Critical Architecture Finding

**Direct HTTP API calls return 503.** All API calls MUST go through `page.evaluate()` in Playwright browser context. Alibaba uses signed requests (`mtop` APIs via `acs.h.alibaba.com`) with appKey, timestamp, and signature — these are auto-generated by the page's JS runtime.

## Auth Tokens (from live session)
- `ctoken` cookie: `_tb_token_` value from browser cookies
- `appKey`: `24889839` (used in mtop API calls — public, same for all buyers)
- `selfAliId`: your Alibaba buyer ID (e.g. `<buyer_id>` — unique per account)
- `selfLoginId`: your Alibaba internal login ID (e.g. `<buyer_login_id>`)
- CSRF token via: `POST //onetalk.alibaba.com/csrf/getToken.htm`
  - Returns: `{"headerName":"X-XSRF-TOKEN","parameterName":"_csrf","token":"..."}`

## Key Identifiers
- `conversationId` format: `{buyerAliId}-{sellerAliId}#11011@icbu`
- `cid` format: `{buyerAliId}-{sellerAliId}#11011` (without @icbu suffix)
- `accountIdEncrypt`: Encrypted seller account ID used in API calls
- `chatToken`: Base64-encoded session token per conversation

## Cross-Origin & Session Notes

### JSONP Same-Origin Requirement
JSONP calls to `mysourcing.alibaba.com` **only work from the same origin**. Cross-origin calls from `message.alibaba.com` fail silently (script loads but callback never fires). Always navigate to `mysourcing.alibaba.com` before making RFQ JSONP calls.

### OTP Validity & Session Expiry
- OTP verification codes appear to remain valid for **~2-3 hours** after receipt
- Browser Use profile cookies expire after **~24 hours** but OTP login is still required
- If sending many OTPs in one session (>5), increase delay between attempts to avoid rate-limiting

### RFQ Page DOM Structure
The RFQ management page at `mysourcing.alibaba.com/rfq/request/rfq_manage_list.htm` contains:
- Container: `.rfq-list-container` (or `.next-table-body` for newer layout)
- Each RFQ card has: date/ID line, subject line, quantity line, quotes count, status
- DOM scraping can be used as a fallback when JSONP fails

## RFQ Posting API (Cloud Browser Required — Discovered Mar 16)

The RFQ posting flow is a multi-step process that requires a fully authenticated cloud browser session on `rfq.alibaba.com`. Direct HTTP calls won't work — CSRF tokens, signed requests, and React state management all require browser context.

### Full Flow

#### Step 1: Upload File
```
POST https://filebroker.alibaba.com/x/upload
Content-Type: multipart/form-data
Body: file=<binary>

Response: {
  "code": 0,
  "filename": "Aba6e5614573d4420a435b6b527c1f368s.xlsx",
  "url": "https://sc01.alicdn.com/kf/Aba6e5614573d4420a435b6b527c1f368s.xlsx",
  "height": null,
  "width": null,
  "size": 6236,
  "hash": "2a8956070bc4c03736769c7c65c64fb1"
}
```
Triggered automatically by Playwright `set_input_files()` on the `input[name="file"]` element.

#### Step 2: Validate File
```
GET https://rfq.alibaba.com/rfq/rfq_annex_check_ajax.do
  ?annexNames={uploaded_filename}
  &ctoken={ctoken}
  &_tb_token_={tb_token}

Response: {"code": 200, "data": ["filename.xlsx"], "success": true}
```

#### Step 3: Save File List (triggered by "Write RFQ details" click)
```
POST https://rfqposting.alibaba.com/rfq/ajax/multimodal/saveFileList.do
  ?ctoken={ctoken}&_tb_token_={tb_token}
Content-Type: application/x-www-form-urlencoded; charset=UTF-8

Body: fileList=[{
  "name": "original_filename.xlsx",
  "fileSavePath": "uploaded_filename.xlsx",
  "downloadURL": "https://sc01.alicdn.com/kf/uploaded_filename.xlsx"
}]

Response: {"status": 200, "success": true, "data": "uuid_memberId"}
```
The `data` field is the `fileKey` used in Step 4.

#### Step 4: Open RFQ Form (popup/new tab)
The "Write RFQ details" button opens a new tab with URL:
```
https://rfq.alibaba.com/rfq/rfqForm.htm
  ?tracelog=rfq-lp-pc
  &newAiForm=true
  &createType=lp_sub
  &subject={url_encoded_text}
  &autoGenerate=true
  &fileKey={uuid_from_step3}
```

#### Step 5: AI Generation (polled automatically)
```
GET https://rfqposting.alibaba.com/rfq/ajax/multimodal/loadResponse.do
  ?requestId={requestId}
  &ctoken={ctoken}
  &_tb_token_={tb_token}

Response (pending): {"status": 200, "success": true}
Response (complete): {
  "status": 200,
  "success": true,
  "data": {
    "rfqTitle": "Custom Printed Food-Grade Stand-Up Pouches...",
    "quantity": 10,
    "quantityUnit": "pieces",
    "detail": "We are seeking two types of food-grade packaging..."
  }
}
```

#### Step 6: Get Biz Code
```
POST https://rfqposting.alibaba.com/rfq/ajax/getBizCode.do
  ?ctoken={ctoken}&_tb_token_={tb_token}

Response: {"status": 200, "success": true, "msg": "success", "data": "icbu_rfq"}
```

#### Step 7: Submit RFQ
Click the `button[type="submit"]` with text "Post request". The form submits with:
- Product name (from AI or manual input)
- Detailed requirements (AI-generated or custom)
- Category (auto-detected: e.g., "Plastic Packaging >> Plastic Bags")
- Sourcing quantity + unit
- Attached files
- Business card sharing consent + terms agreement checkboxes

### Form Fields on rfqForm.htm
| Field | Selector | Required | Notes |
|-------|----------|----------|-------|
| Product name | `input[type="text"]` (first in form) | ✅ | AI-generated from subject |
| Category | `.aigc-post-form-new-container-form-category button` | ✅ | Auto-detected, clickable to change |
| Description | `.aigc-post-form-new-detail-ai-content` | ✅ | AI-generated, editable via "Apply or modify" |
| Quantity | `#quantityInput` | ✅ | Number input |
| Unit | Dropdown next to quantity | ✅ | Default: "pieces" |
| File attachment | Upload area in form | ❌ | Carried over from Step 1 |
| Share Business Card | `input[type="checkbox"]` (1st) | ✅ | Must be checked |
| Terms Agreement | `input[type="checkbox"]` (2nd) | ✅ | Must be checked |

### Key Notes
- The form page at rfqForm.htm is an SPA — no traditional form POST
- AI generation takes 5-30 seconds; poll loadResponse.do until complete
- The `createType=lp_sub` param indicates it was created from the landing page
- `autoGenerate=true` triggers AI generation on page load
- Without `autoGenerate`, the form shows empty fields for manual entry
- File attachment from Step 1 is automatically linked via `fileKey` param

## View Quotes Button Behavior (Discovered Mar 16)

**"View Quotes" opens a popup/new tab** — NOT inline navigation.
- URL pattern: `buyingLeads.htm?hashParams={"rfqId":XXXXX}`
- Popup has two tabs: "My Inquiries" (default) and "My RFQs"
- The `rfqId` param does NOT filter inquiries — all inquiries show for all RFQs
- "My RFQs" tab shows the same RFQ list (recursive — clicking View Quotes opens another popup)

**Playwright handling:** Use `page.expect_popup()` to catch the new tab:
```python
with page.expect_popup(timeout=10000) as popup_info:
    page.locator('.quotation-btn').first.click()
popup = popup_info.value
popup.wait_for_load_state('domcontentloaded')
```

**What "8 Quotes received" really means:**
- These are supplier INQUIRIES (messages), not formal price quotations
- All inquiries go into one pooled inbox, not per-RFQ
- Actual pricing is in the messenger conversation threads
- To extract pricing: use `ali read <n>` on each supplier conversation

**Inquiry DOM structure in popup:**
```
{date}ID: {inquiryId}
Inquiry from TM
{contact name}
{company name}
View inquiry detail
```
